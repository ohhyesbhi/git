`git init` -: Powers your folder to me managed by git , and intialises a new empty repository . It also creates .git folder that has
all the relevant logic to manage versions of your project

`git add <file>` -:  moves file from working area to staging area 

`git remove --cached <file>` -: moves the file back to working area 

`commit` -: Commit is a particualr version of the project . It captures a sanpshot of the project's staged changes and create a 
            version out of it
